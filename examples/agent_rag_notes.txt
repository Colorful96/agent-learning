智能体开发学习笔记

一、当前学习阶段

目前已经完成了 Python 调用大模型 API、提示词设计、结构化输出、输出评测、工具调用、工具注册表、多轮工具循环、短期记忆和长期记忆等内容。

这些内容属于智能体开发的基础工程能力。它们帮助我们理解一个 Agent 不只是调用模型，还需要管理工具、上下文、运行轨迹和错误处理。

二、LangChain 和 LangGraph 的学习顺序

LangChain 更适合先学习。它提供了模型调用、提示词模板、文档加载、检索器、链式调用等常用组件，适合帮助初学者理解 LLM 应用开发的基本模式。

LangGraph 更适合在理解 LangChain 基础之后学习。它强调状态图、节点、边、循环和可控流程，适合构建更复杂的多步骤 Agent。

所以建议先学 LangChain，再学 LangGraph。

三、RAG 的核心概念

RAG 是 Retrieval-Augmented Generation，也就是检索增强生成。它的核心思想不是把所有资料都交给模型，而是先从资料库中检索和问题相关的片段，再让模型基于这些片段回答。

RAG 通常包括文档加载、chunk 切片、embedding 向量化、向量数据库存储、相似度检索、上下文组装和答案生成等步骤。

四、chunk 切片

chunk 是 RAG 中的文本片段。文档通常比较长，不能直接全部放进模型上下文，所以需要切成多个 chunk。

好的 chunk 应该尽量语义完整，比如按标题、段落或句子切分。过大的 chunk 会引入无关信息，过小的 chunk 又可能丢失上下文。

五、embedding 和向量数据库

embedding 是把文本转换成数字向量的过程。语义相近的文本，在向量空间中的距离也更近。

向量数据库用于保存这些文本向量，并在用户提问时检索最相似的文本片段。常见选择包括 Chroma、FAISS、Milvus 等。

六、FastAPI 和部署

当命令行版本的 Agent 跑通以后，可以使用 FastAPI 把它封装成 Web API。

FastAPI 可以提供 HTTP 接口，让前端页面、其他系统或脚本通过网络调用 Agent。部署阶段还需要考虑环境变量、依赖管理、日志、异常处理和接口安全。

七、科研文献智能体目标

最终目标是做一个科研文献智能体。它应该能够读取论文、笔记、Word、PDF 和网页资料，提取重点，生成结构化摘要，回答问题，并给出引用来源。

这个项目可以分阶段完成：先做文本 RAG，再支持 PDF 和 Word，最后增加 Web API 和简单前端。
